🔗 Connexion IA via API

L'extension se connecte à l'API Groq (gratuite) via votre clé personnelle.

  • Créez un compte sur console.groq.com
  • Générez une clé API (gsk_...)
  • Collez-la dans les paramètres de l'extension
  • Vos requêtes transitent directement vers Groq — aucun serveur intermédiaire

🔎 Méthode 1 — Scraping par mot-clé

RECHERCHE Extraction de contenu web par mots-clés

Comment ça marche :

  1. L'utilisateur sélectionne du texte ou clique « Lire la page »
  2. L'extension extrait le contenu visible de la page (sélection → champ actif → formulaires → contenu principal)
  3. Le texte brut est envoyé directement à l'IA avec le prompt de l'agent
  4. L'IA traite le texte et retourne le résultat

✓ PRO

  • Simple et rapide — pas de configuration complexe
  • Fonctionne sur n'importe quelle page web
  • Pas besoin de base de données vectorielle
  • Résultats immédiats, faible latence
  • Léger — pas de dépendances externes

⚠ CON

  • Limité au contenu visible de la page courante
  • Pas de mémoire entre les pages
  • Qualité dépend de la structure HTML de la page
  • Limite de tokens si la page est très longue (troncature à 50K car.)
  • Pas de recherche sémantique — correspondance littérale

🧠 Méthode 2 — Agent RAG

RAG Retrieval-Augmented Generation

Comment ça marche :

  1. Les documents (wiki, KB, procédures) sont découpés en chunks et indexés dans une base vectorielle (embeddings)
  2. Quand l'utilisateur pose une question, une recherche sémantique trouve les chunks les plus pertinents
  3. Les chunks pertinents sont injectés dans le prompt comme contexte
  4. L'IA génère une réponse basée sur ces documents sources

✓ PRO

  • Recherche sémantique — comprend le sens, pas juste les mots
  • Accès à toute une base de connaissances (wiki, docs, procédures)
  • Réponses sourcées et traçables
  • Mise à jour continue de la knowledge base
  • Idéal pour Service Desk avec documentation volumineuse
  • Réduit les hallucinations — l'IA s'appuie sur des faits réels

⚠ CON

  • Nécessite une infrastructure (serveur, base vectorielle, pipeline d'indexation)
  • Configuration initiale plus complexe
  • Coût potentiel (hébergement, API embeddings)
  • Latence plus élevée (recherche + génération)
  • Maintenance de la base de connaissances nécessaire
  • Qualité dépend de la qualité des documents indexés
